# Known Sandbox Errors

This file tracks documentation errors, broken paths, incorrect instructions, and
other issues agents encounter while using this sandbox. It exists so that problems
are recorded immediately rather than rediscovered by the next agent.

**If you are an agent and you encounter an error caused by incorrect documentation,
a missing file, a wrong path, a stale reference, or any other sandbox infrastructure
problem: record it here immediately before continuing your work.** Use the format below.

Do not fix the documentation yourself unless the fix is trivial and obvious. Record
the issue here for human review. Fixing incorrect physics or code is your job; fixing
incorrect sandbox scaffolding is the human's job.

---

## Format

```
### YYYY-MM-DD: Short description
- **Where**: which file/doc had the error
- **What happened**: what you tried and what went wrong
- **Expected**: what should have happened
- **Workaround**: what you did instead (if anything)
```

## Issues

### 2026-04-04: kin_ion.h5 not mentioned in BUILD_INPUTS or run setup docs
- **Where**: `skills/build_inputs/SKILL.md`
- **What happened**: GWJAX run crashed with `FileNotFoundError: kin_ion file not found`
  after completing the full sigma computation. The `kin_ion.h5` file (kinetic+ionic
  matrix elements) is required by gw_jax.py line ~1810 for QP energy output but is
  not listed as a required input artifact in BUILD_INPUTS.md or AGENTS.md manifest
  templates. It's generated by pw2bgw or a separate GWJAX preprocessing step.
- **Expected**: kin_ion.h5 should be listed as a required artifact alongside WFN.h5
  and centroids in the run manifest template and build inputs docs.
- **Workaround**: Symlinked from the reference directory (tests_isdf/mos2_debug_template_1x1/kin_ion.h5).

### 2026-04-04: `use_kih_dat` keyword was a typo — correct flag is `use_kihdat`
- **Where**: `templates/sigma.inp` and `skills/build_inputs/SKILL.md`
- **What happened**: sigma.cplx.x aborts with `ERROR: Unexpected keyword use_kih_dat was found in sigma.inp.` This was NOT a missing feature — the correct BGW flag is `use_kihdat` (no underscore between "kih" and "dat"). Additionally, `dont_use_vxcdat` must also be set to avoid a conflict with the default vxc.dat usage.
- **Fix applied**: Updated `templates/sigma.inp` to use `dont_use_vxcdat` + `use_kihdat`. Updated `skills/build_inputs/SKILL.md` with correct flag names and usage notes.
