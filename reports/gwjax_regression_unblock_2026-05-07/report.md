# GWJAX Regression Unblock

**Date:** 2026-05-07  
**Source tree:** `sources/lorrax_D`  
**Branch:** `agent/cri3-bispinor-repair`

## Summary

The small MoS2 GWJAX regression no longer depends on CUDA-only PHDF5 paths
for CPU execution or on PHDF5 native-library availability in local GPU
developer environments.  The test fixture now runs in a private temporary
directory, writes the current `sigma_diag` output format explicitly, and
compares against a refreshed reference generated by the current driver.

Targeted validation passed on both CPU and GPU.  The broad suite is still
not fully green because of three `test_kmeans_sharded.py` label-tie failures
that are outside this GW regression repair.

## Status

- Done: CPU regression no longer attempts CUDA-only PHDF5 FFI read targets.
- Done: Local GPU regression no longer fails on missing PHDF5 native library dependencies.
- Done: Regression fixture is parallel-safe and does not leave generated outputs in `tests/regression/cohsex_debug/`.
- Done: PHDF5 WFN union reader handles arbitrary nosym `k_ids` order instead of assuming file order.
- Not done: no source commit was made because the broad suite still has unrelated kmeans failures, and `src/gw/ppm_sigma.py` already had pre-existing debug-print edits.

## Open Questions

- The kmeans label mismatches are small-count nearest-centroid disagreements in non-orthogonal cells; they should be triaged separately before this branch is considered globally green.
- The PHDF5 G-space reader remains CUDA-only by design.  CPU production runs should use the fallback path unless a Host PHDF5 FFI implementation is intentionally added later.
