# Si 4×4×4 sigma/BSE — handoff report
**Agent A, 2026-04-24.** All work committed on LORRAX branch `agent-A/ppm-head-guard` (3 commits: `360bde7`, `2655794`, `6aa47b1`).

## Bugs found and fixed (LORRAX source)

1. **PPM bare-Σ^X head was silently dropped** — `gw_jax.py:305` had a `not config.use_ppm_sigma` guard that prevented `static_head_terms` (and therefore the `vhead`/`whead_0freq` overrides) from being computed in PPM mode. Result: PPM `sigX` at Γ band 0 was −14.81 instead of −17.49 eV (missing the 2.6 eV q→0 head). **Fixed:** drop the guard, head now injected for both COHSEX and PPM bare X. Commit `360bde7`.
2. **`use_bgw_vcoul=true` body overlay misplaced BGW values by one BZ** at all q with negative `qvec_wrapped` (37/64 q-points on a 4×4×4 grid). `compute_vcoul.py` mod-wrapped `q_frac` to `[0, 1)` before the `bgw_v_grid_fn` lookup, hiding the integer kg0 connecting LORRAX's signed convention to BGW's `[0, 1)` convention; `find_q_index` then returned `kg0=0` and `unfold_gvecs` left the G-list unshifted. Result: `sigX` dropped from −17.4 → −12.0 eV at Γ band 0 with the overlay on. **Fixed:** drop the `np.mod`. Commit `2655794` (also factors the integer-G transform `G' = S_k·G − kg0` into `common.symmetry_maps.unfold_gvecs` shared between `read_bgw_vcoul.fill_v_grid_for_q` and `SymMaps.get_gvecs_kfull`).
3. **W0_qmunu writeback regression** from commit `4ba05ba` (Apr-12) — fixed previously this session, recommitted on the branch as part of `360bde7`. BSE eigenvalues recovered from ~1.2 eV → 2.47 eV (DFT mode) on Si 4×4×4.

## Open follow-up — required for PPM agreement

**PPM Σ_c head injection is missing.** Commit `1542342` (Apr-10, "Unify head resolution and remove G0 injection") deleted the in-flow `apply_head_correction(V, W, ω)` that injected `v_c0`/`wcoul0` at every requested frequency, and replaced it with `static_head_terms` for COHSEX only. The imag-freq head wiring (`whead_imfreq`, ω=iω_p) was never reinstated. `fit_head_gn`, `compute_head_sigma_at_omega`, and `HeadGNParams` in `head_correction.py` are now dead code (no callers). PPM Σ_c at q=0, G=0 has no head injection at all → ~1–2 eV per-band error vs BGW.

Stop-gap shipped (commit `6aa47b1`): `gw_jax.py` PPM branch now raises `ValueError` if `use_ppm_sigma=true` and any of `vhead`/`whead_0freq` is set without `whead_imfreq`, so the partial-override case is loud instead of silent.

**Real fix still needed:** plumb a static-head-style injection through the PPM Σ_c path. The mechanism is the same as the COHSEX `_add_head` — compute a Σ_c_head per occupied band from `vhead` + `whead_imfreq` (and whatever the PPM ω-integration convention requires), inject after the Σ_c(ω) construction in `ppm_sigma.py`. The dead-code `compute_head_sigma_at_omega` in `head_correction.py` is roughly the right shape; reanimating it (or writing the equivalent against the new `static_head_terms` API) is the path forward.
