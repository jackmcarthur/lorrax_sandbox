# GWJAX `cohsex.in` Input Reference

GWJAX is the GW module of **LORRAX** (**Lo**w-scaling **R**eal-space **R**eal-**A**xis
e**X**cited state package). This is the exhaustive reference for every parameter
in the GWJAX input file. All parameters
are read by `gw_init.read_cohsex_input()` from a `[cohsex]` INI section.

---

## 1. Restart & ISDF Basis

### `restart` (bool, default: `true`)
If true, load precomputed ISDF tensors (V_qŒºŒΩ, wavefunctions, centroids) from
`tmp/isdf_tensors_<N>.h5`. If false, recompute from scratch (slow: FFT + Cholesky
fitting of zeta_q). Must be false on first run or when `nband` changes.

**Restart file contents** (`tmp/isdf_tensors_<N>.h5`):
| Dataset | Shape | Type | Meaning |
|---------|-------|------|---------|
| `psi_full_y` | (nk, nband, nspinor, n_rmu) | complex128 | Wavefunctions at ISDF centroids (Y-layout) |
| `enk_full` | (nk, nband) | float64 | DFT eigenvalues (Ry) |
| `V0_noG0_munu` | (n_rmu, n_rmu) | complex128 | Bare Coulomb V(q=0) without G=0 head |
| `G0_mu_nu` | (n_rmu,) | complex128 | G=0 centroid overlap vector Œ∂_Œº(G=0) |

### `centroids_file` (str, default: `"centroids_frac.txt"`)
Path to ISDF centroid coordinates in fractional crystal coordinates. One line per
centroid (3 floats). Generated by `python -m centroid.kmeans_isdf`. The number
of centroids N_Œº determines the ISDF basis size; recommended 8√ó(nval+ncond) for
debugging, 12√ó for production.

### `use_chunked_isdf` (bool, default: `true`)
Use memory-efficient chunked ISDF fitting (processes r-space in chunks). Required
for large systems where the full FFT grid doesn't fit in GPU memory.

### `memory_per_device_gb` (float, default: `0.0`)
GPU memory budget in GB. 0.0 = auto-detect (90% of available GPU memory via
`jax.memory_stats()` or `nvidia-smi`). The chunk solver uses this to determine
r-chunk, band-chunk, and q-chunk sizes. See `MEMORY_MODEL.md` for formulas.

### `zeta_ridge_eps` (float, default: `0.0`) ‚Äî opt-in, charge-only
Relative Tikhonov ridge for the Œ∂-fit. `0.0` (default) = OFF, bit-identical to
the historical LSQ fit. When `> 0` the fit solves the SPD normal equations
`(C¬≤ + Œµ_q¬≤I) Œ∂ = C Z` with `Œµ_q = zeta_ridge_eps ¬∑ ŒªÃÇ_max(C_q)` per q
(deterministic power-iteration ŒªÃÇ, no eigh) ‚Äî the spectral filter
`f_Œµ(Œª) = Œª/(Œª¬≤+Œµ¬≤)`, i.e. the "cleaned Œ∂" of the F-scheme study. RELATIVE
scaling by design so the knob transfers across system sizes. Charge channel
only: rejected with `bispinor = true` (transverse indefinite-CCT follow-up).
Effective solver conditioning: `cond(C¬≤+Œµ¬≤I) ‚âà min(cond(C)¬≤, 1/eps_rel¬≤)` ‚Äî
the crossover Œµ_rel* = cond(C)^{-1/2} (~1e-4 at fixture scale). A/B results
(reports/zeta_ridge_ab_2026-07-17/): NOT physically inert on GW Œ£ ‚Äî drifts
are ~1e-3 relative of Œ£ (meV‚Äì100 meV absolute); V/W-tile q-covariance
improves ~8√ó at 1e-4 on Si. Diagnostic/experiments knob; do not enable in
production comparisons without an A/B against `zeta_ridge_eps = 0`.

### `charge_zeta_solve` (str, default: `rank_truncate`) ‚Äî charge-only
Selects how the charge ISDF Œ∂-fit factors its (identity-padded, replicated,
mesh-invariant) CCT `C_q`:
- **`rank_truncate`** (default) ‚Äî rank-revealing `eigh` pseudo-inverse: form
  `B = V¬∑diag(1/‚àöŒª_keep)` keeping only `Œª > zeta_rcond¬∑Œª_max`, so `B B·¥¥ = C‚Å∫`
  (truncated pseudo-inverse) and `Œ∂ = C‚Å∫Z = B(B·¥¥Z)`. DROPS the near-null
  directions that appear when n_Œº over-completes the pair-density rank (Œ∫~1e13),
  which plain Cholesky otherwise amplifies into O(1) V_q errors ‚Üí tens-of-eV
  GN-PPM Œ£_c blow-up (device/nband-dependent, gap-inverting). This is the
  principled cure for the conduction-Œ£c catastrophe.
- **`cholesky`** ‚Äî the historical replicated / cuSolverMp Cholesky path,
  **bit-identical to pre-2026-07 behavior**; the selectable alternative.

Well-conditioned CCTs (n_Œº ‚â§ pair-density rank) are unaffected: rank_truncate
drops ~0 modes and equals cholesky within ~1e-13. Over-complete bases diverge ‚Äî
that is the point. Cross-code parity: MoS‚ÇÇ 4√ó4 GN-PPM vs BGW, over-complete
n_Œº=1204 ‚Üí Cholesky MAE 3.0 eV/max 12.96 eV ‚Üí rank_truncate MAE 0.038 eV/max
0.077 eV across the whole `zeta_rcond` = 1e-8‚Ä¶1e-4 plateau.
See `reports/gw_rank_truncation_2026-07-20/`.

### `zeta_rcond` (float, default: `1e-8`) ‚Äî charge-only, `rank_truncate` cutoff
Relative eigenvalue cutoff for `charge_zeta_solve = rank_truncate`: drop
eigenvalues `Œª < zeta_rcond ¬∑ Œª_max(C_q)` (per q). Env override
`LORRAX_ZETA_RCOND`.

The cure an over-complete basis needs is a **plateau**, not a point ‚Äî so the
default sits at the plateau's LOW end, because truncation is cheap but **not
free** on a well-conditioned basis:
- **Over-complete** (MoS‚ÇÇ 4√ó4 / 1204c): `1e-10` under-truncates and only
  partially recovers (MAE 1.4 eV vs BGW); `1e-8`, `1e-6` and `1e-4` all collapse
  to ~0.04 eV. The recovery plateau spans `1e-8 ‚Ä¶ 1e-4`; `1e-4` starts
  over-truncating at its top end.
- **Well-conditioned** (MoS‚ÇÇ 4√ó4 / 640c): drops ~0 modes, equals `cholesky` to
  ~1e-13.
- **Well-conditioned but not truncation-free** (bulk Si 4√ó4√ó4 / 960c, the
  BGW-anchored `si_cohsex_3d` gate): this spectrum *does* extend below
  `1e-6¬∑Œª_max`. Measured max |Œî| of `sigTOT` vs the `1e-10` freeze
  (`reports/gw_conduction_postfix_2026-07-21/si_rcond_sweep.sh`):

  | rcond | 1e-9 | 1e-8 | 1e-7 | 1e-6 | 1e-5 | 1e-4 |
  |---|---|---|---|---|---|---|
  | max \|Œî sigTOT\| (meV) | 0.001 | **0.054** | 0.417 | 1.021 | 2.918 | 37.2 |

  `1e-8` is the plateau's bottom: identical over-complete cure, ~20√ó less drift
  on the BGW anchor than `1e-6` (0.054 vs 1.021 meV). That 0.054 meV is well
  inside Si's 0.48 meV max \|Œî\| vs BerkeleyGW, so the `si_cohsex_3d` fixture
  no longer needs to pin `zeta_rcond` away from production ‚Äî it runs at the
  default and the gate therefore exercises the shipping code path.

Ignored for `charge_zeta_solve = cholesky`. See
`reports/gw_rank_truncation_2026-07-20/`,
`reports/gw_conduction_postfix_2026-07-21/` and
`reports/gw_bandrange_centroids_2026-07-21/`.

## 2. Band Window

The GW calculation partitions bands into windows defined by four edges b0..b4.
The number of electrons `nelec` is read from the WFN file.

### `nval` (int, default: `5`)
Number of **valence** (occupied) bands in the sigma evaluation window. Sets
`b1 = nelec ‚àí nval`. These are the highest occupied bands for which Œ£ is computed.

### `ncond` (int, default: `5`)
Number of **conduction** (unoccupied) bands in the sigma evaluation window. Sets
`b3 = nelec + ncond`. These are the lowest unoccupied bands for which Œ£ is computed.

### `nband` (int, default: `100`)
Total number of bands loaded from the WFN for screening (œá‚ÇÄ) and self-energy sums.
Sets `b4 = nband`. Must match `number_bands` in BGW's epsilon.inp and sigma.inp
for fair comparison. The WFN file must contain at least this many bands.

**Band edge summary** (all 0-indexed):
```
b0 = 0          (bottom of full range)
b1 = nelec ‚àí nval   (bottom of sigma window, within valence)
b2 = nelec          (Fermi level: top of occupied)
b3 = nelec + ncond  (top of sigma window, within conduction)
b4 = nband          (top of full range for screening)
```

**Wavefunction slices** built from these:
| Slice | Range | Used for |
|-------|-------|----------|
| `v_slice` | b0 : b2 | Valence (occupied) bands in œá‚ÇÄ |
| `c_slice` | b2 : b4 | Conduction (unoccupied) bands in œá‚ÇÄ |
| `l_slice` | b0 : b3 | Sigma window bands (for SX Green's function) |
| `coh_slice` | b0 : b4 | All bands (for COH resolution of identity and PPM G) |

### `sys_dim` (int, default: `2`)
System dimensionality controlling the Coulomb truncation:
- **0**: Molecule (Wigner-Seitz box truncation). V(G=0) is finite. The q=0 head
  of W equals V (no mini-BZ averaging needed).
- **2**: 2D slab (cell slab truncation). V(q+G) = 8œÄ/|q+G|¬≤ √ó (1 ‚àí e^{‚àíz_c|q+G|_xy} cos(q_z z_c))
  divided by cell volume. V(G=0) is set to zero; the head is injected separately
  via mini-BZ-averaged wcoul0.

Corresponds to BGW's `cell_box_truncation` (0) or `cell_slab_truncation` (2).

## 3. Calculation Mode

### `x_only` (bool, default: `false`)
If true, compute only bare exchange Œ£_X = ‚àí(1/N_k) Œ£_q G_{occ,k‚àíq} V_q. No
screening, no correlation. Mutually exclusive with `do_screened = true`.

### `do_screened` (bool, default: `true`)
If true, compute the screened Coulomb interaction W from œá‚ÇÄ via the Dyson equation
W = V(I ‚àí Vœá‚ÇÄ)‚Åª¬π. The static COHSEX self-energy is then:
```
Œ£_SX = ‚àí(1/N_k) Œ£_q G_{occ,k‚àíq} W_q
Œ£_COH = ‚àí(1/2N_k) Œ£_q I_{k‚àíq} (W_q ‚àí V_q)
```
If false, W = V (bare exchange only).

### `bispinor` (bool, default: `false`)
If true, use 4-component Dirac bispinor wavefunctions with Œ≥‚Å∞ Coulomb vertex
factors: Œ£_{ab} = Œ≥‚Å∞_{aa} Œ≥‚Å∞_{bb} G_{ab} V, where Œ≥‚Å∞ = diag(1,1,‚àí1,‚àí1).
If false, use 2-component Pauli spinors (standard for spin-orbit coupling).

### `qp_solver` (str, default: `"auto"` ‚Üí `"one_shot_dft"`)
How QP energies are extracted from Œ£ ‚Äî orthogonal to the Œ£ ansatz
(`compute_mode`). Three states, each naming a standard method:

- **`"one_shot_dft"`** (the default): textbook G0W0. Œ£ is built once from the
  DFT inputs and *everything* is evaluated at E_DFT ‚Äî the eqp0/eqp1 text
  outputs (at-DFT Newton + Z-linearization, as always) AND the
  QSGW-symmetrised Œ£_xc whose eigh produces `E_qp_ry` /
  `qp_wfn_rotations.h5` / `WFN_qp.h5`. No iteration of any kind.
- **`"fixed_point"`**: one-shot Œ£ + diagonal on-shell solve
  E = h0 + Re Œ£(E) for the QSGW-build evaluation energies (eigenvalue-only;
  Œ£ is never rebuilt). Dynamic modes (`gn_ppm`/`hl_ppm`) only ‚Äî rejected at
  config validation for static modes (no œâ-grid to solve on).
  `sigma_at_dft_extrapolate` is a sub-knob of this state.
- **`"self_consistent"`**: full QSGW loop ‚Äî Œ£ rebuilt each iteration from
  rotated œà + the previous iteration's E (Œ∂/V_q are *not* refit; œá‚ÇÄ‚ÜíW is
  re-solved every iteration). Works for all modes (COHSEX and GN-PPM
  verified end-to-end). Loop knobs: the `sc_*` keys below.

`"auto"` resolves: `self_consistent = true` ‚Üí `"self_consistent"`, else
`"one_shot_dft"`. Validation: `fixed_point`/`self_consistent` √ó
`sigma_omega_accumulation = kij_stream` is rejected (streamed Œ£_c(œâ) leaves
no in-memory œâ-tensor for the on-shell solve / QSGW rebuild).

eqp0.dat / eqp1.dat keep the same formula in all three states; only the
provenance of Œ£ changes under `self_consistent` (converged Œ£, still
evaluated at E_DFT).

### `self_consistent` (bool, default: `false`) ‚Äî DEPRECATED
Deprecated alias (2026-07-08) for `qp_solver = self_consistent`; still
honored via `qp_solver = auto` resolution. Set `qp_solver` instead.

### `sc_max_iter` (int, default: `20`)
Maximum self-consistency iterations (`qp_solver = self_consistent` only).
The deprecated `LORRAX_SC_MAX_ITER` env var overrides it (a note is printed).

### `sc_tol_ev` (float, default: `1e-4`)
Convergence threshold on the RMS ŒîE (eV) between consecutive iterations'
QP eigenvalues. Env override: `LORRAX_SC_TOL_EV`.

### `sc_accelerator` (str, default: `"rcrop"`)
- **`"rcrop"`**: Anderson-style restart-CROP acceleration ‚Äî required for QSGW
  on dense band manifolds (the fixed-point Jacobian typically has a 2-cycle
  direction). Makes TWO pipeline calls per accelerator iteration
  (trial + residual).
- **`"linear"`**: plain Œ±-mixing with damping `sc_mixing` (diagnostic).

Env override: `LORRAX_SC_ACCEL`.

### `sc_history_depth` (int, default: `5`)
rCROP history depth (m=5 is BGW's QSGW default). Env override:
`LORRAX_SC_DEPTH`.

### `sc_mixing` (float, default: `1.0`)
Linear damping coefficient Œ± (`sc_accelerator = linear` only). Env override:
`LORRAX_SC_MIXING`.

### `sc_dump_dir` (str, default: `""`)
If set, dump the per-iteration QP-energy history to
`<sc_dump_dir>/e_history_kn_ev.npy`. Env override: `LORRAX_SC_DUMP_DIR`.

## 4. Screening

### `screening_method` (str, default: `"minimax"`)
Method for computing œá‚ÇÄ ‚Üí W:
- **`"minimax"`**: Laplace minimax quadrature for the imaginary-time representation
  of œá‚ÇÄ. Approximates 1/(E_c ‚àí E_v) ‚âà Œ£_l w_l exp(‚àíœÑ_l (E_c ‚àí E_v)) with O(ln R)
  nodes where R = E_bw/E_gap. This is the production method.

### `minimax_target_error` (float, default: `1e-6`)
L‚àû error target for the minimax rational approximation of 1/x on [E_gap, E_bw].

### `minimax_max_nodes` (int, default: `64`)
Maximum number of quadrature nodes for the minimax fit.

### `minimax_energy_reference` (str, default: `"midgap"`)
Energy reference for the minimax window construction. Options: `"midgap"`, `"vbm"`,
or a numeric value in Ry. Shifts the energy origin to center the minimax interval,
improving numerical stability.

## 5. Finite-Size (Head) Corrections

For periodic systems (sys_dim ‚â• 1), the q=0 Coulomb divergence is handled by:
1. Setting V(G=0) = 0 in the body
2. Computing mini-BZ-averaged heads `‚ü®v(q)‚ü©` and `‚ü®W(q)‚ü© = ‚ü®v/(1‚àív¬∑q^T S q)‚ü©`
3. Adding the head as a rank-1 correction: `(wcoul0/Œ©) |Œ∂(G=0)‚ü©‚ü®Œ∂(G=0)|`

### `wcoul0_source` (str, default: `"s_tensor"`)
Source for the q‚Üí0 head values:
- **`"s_tensor"`** (default): Compute the frequency-dependent S(œâ) tensor from
  dipole matrix elements (dipole.h5), then evaluate `wcoul0(œâ) = ‚ü®v/(1‚àív¬∑q^T S(œâ) q)‚ü©`
  via Sobol QMC on the mini-BZ. This gives frequency-dependent heads, which is
  required for correct GN-PPM extraction at both œâ=0 and œâ=iœâ_p.
- **`"epshead"`**: Use the static Œµ‚Åª¬π(G=0,G'=0) from eps0mat.h5. Static only ‚Äî
  the same head is used at all frequencies. **Debug only; gives wrong PPM at œâ‚âÝ0.**

### `vhead` (float, default: `None`)
Override the bare Coulomb head v(q‚Üí0) in atomic units (same as printed under
`FINITE-SIZE CORRECTIONS` in the output). If set together with `whead_0freq`,
bypasses `determine_wcoul0()` entirely. Useful for injecting BGW epsilon.out
head values for debugging.

### `whead_0freq` (float, default: `None`)
Override the screened Coulomb head W(q‚Üí0, œâ=0) in atomic units.

### `whead_imfreq` (float, default: `None`)
Override the screened Coulomb head W(q‚Üí0, œâ=iœâ_p) in atomic units. Only used
when `use_ppm_sigma = true`. Must be set together with `vhead` and `whead_0freq`.

### `head_minibz_average` (bool, default: `false`)
Per-Q mini-BZ Coulomb-head **cell averaging** (BGW `minibzaverage_3d`/`_2d`,
`Common/minibzaverage.f90`). Controls whether the Coulomb head is a POINT value
or a **mini-BZ cell average** `‚ü®v(q+G)‚ü©_mBZ` (the average of the bare Coulomb over
the q-grid's Voronoi cell).

- **`false`** (default): the historical behavior, **bit-identical**. The q‚Üí0 head
  is the pure-Sobol mini-BZ mean and every **finite-Q** exchange head (the
  arbitrary-Q BSE `V_Q` in `bse.exciton_bands`) is the analytic POINT value
  `v(Q+G*)`.
- **`true`**: routes the head through the single-source
  `gw.coulomb.base.minibz_average`:
  - the q‚Üí0 **3D** head gains the analytic **Baldereschi-Tosatti sphere term**
    (`32œÄ¬≤‚àöq0sph2/V_mBZ`), which makes it **seed-independent** (the pure-Sobol
    mean drifts between seeds from the few tiny `Œ¥q ‚Üí 8œÄ/|Œ¥q|¬≤` blow-ups);
  - the Voronoi fold widens (`nmax 1‚Üí3`, BGW `ncell=3`) for skewed cells;
  - the BSE arbitrary-Q `eval_vq` head becomes the mini-BZ **cell average**
    `‚ü®v_LR(Q+G*)‚ü©_mBZ`, fixing the **4‚Äì13 % near-Œì / zone-boundary**
    point-vs-cell-average error (arbitrary_q_bse.md ¬ß16.4). Only the LR channel
    is averaged; the SR body carries `v_SR(Q+G*)` once (no double-count, ¬ß16).

The winding is unaffected: for 2D slabs the head is a `|Q|` cusp, and the
directional `e^{-i2Œ∏}` structure rides the phase-factored Œ∂ÃÉ rank-1 term ‚Äî the
cell average supplies only the head **magnitude**. The BSE driver
`bse.exciton_bands` also exposes `--head-minibz-average` to override this key
per-run.

### `bse_k_grid` (str, default: `""` ‚Äî off)

Fine-grid **densification** of the ENTIRE BSE problem. Set to `"NX NY NZ"` (or
`"NX,NY,NZ"`) and the GENERAL BSE init
(`bse.bse_io.load_bse_data_from_restart_sharded`, the single choke point where
every BSE solver obtains its `data` bundle) interpolates the whole coarse-grid
problem onto that finer k-grid **before any solve**, so **every** BSE solver
(`exciton_bands`, `bse_feast`, `bse_nontda`, `bse_kpm`, the W-resolvent) runs on
the fine grid transparently ‚Äî no per-driver flag. What is interpolated:

- **œà_{v,c}(k)** and **QP Œµ_{v,c}(k)** ‚Äî one htransform fH
  (`bandstructure.bse_setup.compute_wfns_fi` with `kgrid_fi = NX NY NZ`), the
  same interpolator `exciton_bands` uses for its Q-path caches.
- **V_Q exchange** ‚Äî `bse.vq_interp.build_vq_evaluator` (prepared ONCE on the
  coarse grid) evaluated at Q=0 with the **fine** mini-BZ head. A Q=0 exciton's
  exchange is the single q=0 tile (dense in k,k'), so the fine grid's exchange
  q-set is just q=0.
- **W direct** ‚Äî `bse_io.pad_W_R_to_grid` zero-pads the coarse `W_R` onto the
  fine k-lattice (exact trigonometric interpolation through the coarse
  samples), fft'd back to a fine `W_q`. This subsumes the `exciton_bands`
  `--w-coarse-grid` W-only flag: `bse_k_grid` drives the pad automatically.

Each fine length must be a **positive multiple** of the matching coarse restart
length (coarse BZ ‚äÇ fine BZ). **Empty (default) or == the coarse grid ‚Üí the
coarse `data` bundle is returned byte-identically** (the fast path is untouched;
band/Œº dims and the q=0 head projectors are grid-invariant, only the k axis
grows).

Accuracy caveat: the htransform œà/Œµ recovery degrades if the loaded band window
over-packs the centroid basis ‚Äî keep `nband` **modest** (‚â§ ~48 for MoS2/640c;
`10_lorrax_exciton_bands_80interp_8v8c`), a few conduction guard bands above the
BSE window, not maximal. The exchange head uses `vq_interp`'s mini-BZ LR-head
convention (as in `exciton_bands`), which differs from the loader's GW-computed
`vhead` rank-1 on a native fine restart; the lowest exciton is direct-term
dominated, so this is a secondary effect. Slab (2D) only ‚Äî the `vq_interp`
exchange path asserts `q_z = 0`, same restriction as `exciton_bands`.

## 6. GN-PPM Dynamic Self-Energy

When `use_ppm_sigma = true`, the code computes the full frequency-dependent
correlation self-energy Œ£^c(œâ) using the Godby-Needs Generalized Plasmon Pole
model. The PPM represents W^c(œâ) = W(œâ) ‚àí V as:

```
W^c_q(Œº,ŒΩ,œâ) = 2 B_q(Œº,ŒΩ) Œ©_q(Œº,ŒΩ) / (œâ¬≤ ‚àí Œ©_q¬≤(Œº,ŒΩ))
```

where B and Œ© are extracted elementwise from W^c(0) and W^c(iœâ_p) via:
```
Œ© = œâ_p ‚àö(W^c(iœâ_p) / (W^c(0) ‚àí W^c(iœâ_p)))
B = ‚àí¬Ω W^c(0) Œ©
```

The self-energy is then evaluated via minimax-windowed CTSP (Complex-Time
Shredded Propagator) integration in imaginary time, decomposed into two branches:
```
Œ£‚Å∫(œâ) = ‚àí(1/N_k) Œ£_{v,q} M¬≤ B/(œâ + h_v + Œ©)    [occupied/valence branch]
Œ£‚Åª(œâ) = ‚àí(1/N_k) Œ£_{c,q} M¬≤ B/(œâ ‚àí E_c ‚àí Œ©)    [unoccupied/conduction branch]
```

### `use_ppm_sigma` (bool, default: `false`)
Enable the GN-PPM dynamic Œ£^c(œâ) pipeline. Requires `do_screened = true`.
When enabled, replaces the static COH with the frequency-integrated result.

### `ppm_omega_p` (float, default: `2.0` Ry)
Imaginary frequency probe for the GN-PPM extraction. W is evaluated at œâ=0
and œâ=iœâ_p to determine the PPM parameters. Must match the second frequency
in BGW's epsilon (which is 2.0 Ry = 27.2 eV by default).

### `ppm_fallback_omega` (float, default: `2.0` Ry)
Fallback plasmon frequency for GN modes where the extraction gives unphysical
(negative or imaginary) Œ©. The fitted pole is only used when
`ppm_invalid_mode = "2ry"`, but the fit always bakes it in (`B = ‚àí¬Ω¬∑W^c(0)¬∑Œ©`).

### `ppm_invalid_mode` (str, default: `"static_limit"`)
Policy for invalid GN modes (Œ©¬≤ < 0), = BGW `invalid_gpp_mode`
(implemented 2026-07-08; validated vs BGW modes 0/2/3 on Si 4√ó4√ó4 ‚Äî
see `reports/bgw_invalid_mode_refs_2026-07-08/`):
- **`"static_limit"`** (BGW mode 3 = BGW default): exclude invalid modes from
  the dynamic pole sum and add the analytic œâ-independent static-COHSEX term
  Œ£_invalid = Œ£_occ(G_occ, W^c_inv(0)) ‚àí ¬Ω Œ£_RI(I, W^c_inv(0)).
- **`"zero"`** (BGW mode 0): drop invalid modes entirely.
- **`"2ry"`** (BGW mode 2): keep the fit's `ppm_fallback_omega` pole.
- `"infinity"` is accepted as an alias of `"static_limit"`; `"imaginary"`
  (BGW mode 1) is unsupported.

### `fermi_reference` (str, default: `"midgap"`)
Reference energy for the œâ grid and the Œ£¬± branch splitting:
- **`"midgap"`**: E_F = (VBM + CBM)/2. Band energies become E_c = E_{ck} ‚àí E_F ‚â• 0
  and h_v = E_F ‚àí E_{vk} ‚â• 0. The œâ grid is centered on E_F.
- **`"vbm"`**: E_F = VBM.

## 7. Frequency Grid & Windowing

### `sigma_omega_min_ev` (float, default: `-5.0`)
Lower bound of the Œ£^c(œâ) output grid in eV (relative to E_F per `fermi_reference`).

### `sigma_omega_max_ev` (float, default: `5.0`)
Upper bound of the Œ£^c(œâ) output grid in eV.

### `sigma_omega_step_ev` (float, default: `0.25`)
Spacing of the output frequency grid. n_œâ = floor((max ‚àí min)/step) + 1.

### `sigma_regularization_ev` (float, default: `0.25`)
Regularization width Œæ (eV) for crossing windows. Controls the sine-sum
approximation to 1/x near the singularity.

The minimax method partitions the denominators of G and W into multiple windows
with different quadrature procedures:

- **Non-crossing (sign-definite) windows**: The denominator `S ¬± œâ` doesn't
  change sign within the window. Approximated by exponential sums (Laplace
  minimax): `1/x ‚âà Œ£_l Œ±_l exp(‚àíœÑ_l x)`. Cost: O(ln R) nodes where R is
  the dynamic range of the window.

- **Crossing windows**: The denominator passes through zero for some (band, œâ)
  combinations. Approximated by sine sums with a regularization envelope:
  `G_reg(u) ‚âà Œ£_l w_l sin(œÑ_l u)`, where `u = (S‚àíœâ)/Œæ` is the dimensionless
  energy scaled by the regularization width Œæ. This smooths the 1/x singularity
  over a width ~Œæ, analogous to iŒ∑ broadening. The number of sine-sum nodes
  scales as O(A) where A = (sigma_max ‚àí sigma_min) / Œæ ‚Äî **linearly proportional
  to the frequency range divided by the regularization width**. Smaller Œæ gives
  a narrower broadening but requires more quadrature nodes and is more expensive.

- **Three-window scheme**: For the conduction branch at œâ > 0, the code
  partitions (E_A, E_B) space into a core (crossing, sine-sum), a_stripe
  (sign-definite, Laplace), and b_slab (sign-definite, Laplace). Only the
  core uses the expensive sine-sum quadrature.

Converted to Ry internally: Œæ_Ry = Œæ_eV / 13.606.

### `ppm_sigma_scale` (float, default: `1.0`)
Global multiplicative scale factor for Œ£^c. For debugging/empirical adjustment.

### `ppm_sigma_flip_neg` (bool, default: `false`)
**Debug only.** If true, flip the sign of the œâ < E_F block. Test parameter for
sign convention debugging.

## 8. Accumulation & Evaluation at E_DFT

### `sigma_omega_accumulation` (str, default: `"auto"`)
How to accumulate Œ£_c(k,i,j,œâ) over time nodes:
- **`"auto"`**: Choose `"kij"` (in-memory) or `"kij_stream"` based on memory.
- **`"kij"`**: Accumulate the full (n_œâ, nk, nb, nb) tensor in GPU memory. Fast

- **`"kij_stream"`**: Stream œâ-chunks to `sigma_kij_h5_file` on disk. For very

### `sigma_at_dft_energies` (bool, default: `false`) ‚Äî DEPRECATED

## 11. K-Points

**The k-grid for the GW calculation is taken entirely from the WFN.h5 file**
(nkx √ó nky √ó nkz stored in its header). The K_POINTS block in cohsex.in does
**NOT** control the GW k-grid. Changing the K_POINTS block does not change which
k-points are used for screening, self-energy, or any physical quantity.

The K_POINTS block is used **only** for band-structure interpolation plots (the
QP energy plot output). It follows Quantum ESPRESSO `{crystal_b}` conventions:

```
K_POINTS {crystal_b}
4
  0.0000 0.0000 0.0000 80  #Gamma
  0.0000 0.5000 0.0000 80  #M
  0.3333 0.3333 0.0000 80  #K
  0.0000 0.0000 0.0000 80  #Gamma
```

The integer after each k-point (80) specifies the number of interpolation points
along each segment for the band-structure plot. It is not used by the GW engine.
