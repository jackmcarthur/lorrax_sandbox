# GWJAX `cohsex.in` Input Reference

GWJAX is the GW module of **LORRAX** (**Lo**w-scaling **R**eal-space **R**eal-**A**xis
e**X**cited state package). This is the exhaustive reference for every parameter
in the GWJAX input file. All parameters
are read by `gw_init.read_cohsex_input()` from a `[cohsex]` INI section.

---

## 1. Restart & ISDF Basis

### `restart` (bool, default: `true`)
If true, load precomputed ISDF tensors (V_qОјОЅ, wavefunctions, centroids) from
`tmp/isdf_tensors_<N>.h5`. If false, recompute from scratch (slow: FFT + Cholesky
fitting of zeta_q). Must be false on first run or when `nband` changes.

**Restart file contents** (`tmp/isdf_tensors_<N>.h5`):
| Dataset | Shape | Type | Meaning |
|---------|-------|------|---------|
| `psi_full_y` | (nk, nband, nspinor, n_rmu) | complex128 | Wavefunctions at ISDF centroids (Y-layout) |
| `enk_full` | (nk, nband) | float64 | DFT eigenvalues (Ry) |
| `V0_noG0_munu` | (n_rmu, n_rmu) | complex128 | Bare Coulomb V(q=0) without G=0 head |
| `G0_mu_nu` | (n_rmu,) | complex128 | G=0 centroid overlap vector О¶_Ој(G=0) |

### `centroids_file` (str, default: `"centroids_frac.txt"`)
Path to ISDF centroid coordinates in fractional crystal coordinates. One line per
centroid (3 floats). Generated by `python -m centroid.kmeans_isdf`. The number
of centroids N_Ој determines the ISDF basis size; recommended 8Г—(nval+ncond) for
debugging, 12Г— for production.

### `use_chunked_isdf` (bool, default: `true`)
Use memory-efficient chunked ISDF fitting (processes r-space in chunks). Required
for large systems where the full FFT grid doesn't fit in GPU memory.

### `memory_per_device_gb` (float, default: `0.0`)
GPU memory budget in GB. 0.0 = auto-detect (90% of available GPU memory via
`jax.memory_stats()` or `nvidia-smi`). The chunk solver uses this to determine
r-chunk, band-chunk, and q-chunk sizes. See `MEMORY_MODEL.md` for formulas.

## 2. Band Window

The GW calculation partitions bands into windows defined by four edges b0..b4.
The number of electrons `nelec` is read from the WFN file.

### `nval` (int, default: `5`)
Number of **valence** (occupied) bands in the sigma evaluation window. Sets
`b1 = nelec в€’ nval`. These are the highest occupied bands for which ОЈ is computed.

### `ncond` (int, default: `5`)
Number of **conduction** (unoccupied) bands in the sigma evaluation window. Sets
`b3 = nelec + ncond`. These are the lowest unoccupied bands for which ОЈ is computed.

### `nband` (int, default: `100`)
Total number of bands loaded from the WFN for screening (П‡в‚Ђ) and self-energy sums.
Sets `b4 = nband`. Must match `number_bands` in BGW's epsilon.inp and sigma.inp
for fair comparison. The WFN file must contain at least this many bands.

**Band edge summary** (all 0-indexed):
```
b0 = 0          (bottom of full range)
b1 = nelec в€’ nval   (bottom of sigma window, within valence)
b2 = nelec          (Fermi level: top of occupied)
b3 = nelec + ncond  (top of sigma window, within conduction)
b4 = nband          (top of full range for screening)
```

**Wavefunction slices** built from these:
| Slice | Range | Used for |
|-------|-------|----------|
| `v_slice` | b0 : b2 | Valence (occupied) bands in П‡в‚Ђ |
| `c_slice` | b2 : b4 | Conduction (unoccupied) bands in П‡в‚Ђ |
| `l_slice` | b0 : b3 | Sigma window bands (for SX Green's function) |
| `coh_slice` | b0 : b4 | All bands (for COH resolution of identity and PPM G) |

### `sys_dim` (int, default: `2`)
System dimensionality controlling the Coulomb truncation:
- **0**: Molecule (Wigner-Seitz box truncation). V(G=0) is finite. The q=0 head
  of W equals V (no mini-BZ averaging needed).
- **2**: 2D slab (cell slab truncation). V(q+G) = 8ПЂ/|q+G|ВІ Г— (1 в€’ e^{в€’z_c|q+G|_xy} cos(q_z z_c))
  divided by cell volume. V(G=0) is set to zero; the head is injected separately
  via mini-BZ-averaged wcoul0.

Corresponds to BGW's `cell_box_truncation` (0) or `cell_slab_truncation` (2).

## 3. Calculation Mode

### `x_only` (bool, default: `false`)
If true, compute only bare exchange ОЈ_X = в€’(1/N_k) ОЈ_q G_{occ,kв€’q} V_q. No
screening, no correlation. Mutually exclusive with `do_screened = true`.

### `do_screened` (bool, default: `true`)
If true, compute the screened Coulomb interaction W from П‡в‚Ђ via the Dyson equation
W = V(I в€’ VП‡в‚Ђ)вЃ»В№. The static COHSEX self-energy is then:
```
ОЈ_SX = в€’(1/N_k) ОЈ_q G_{occ,kв€’q} W_q
ОЈ_COH = в€’(1/2N_k) ОЈ_q I_{kв€’q} (W_q в€’ V_q)
```
If false, W = V (bare exchange only).

### `bispinor` (bool, default: `false`)
If true, use 4-component Dirac bispinor wavefunctions with ОівЃ° Coulomb vertex
factors: ОЈ_{ab} = ОівЃ°_{aa} ОівЃ°_{bb} G_{ab} V, where ОівЃ° = diag(1,1,в€’1,в€’1).
If false, use 2-component Pauli spinors (standard for spin-orbit coupling).

### `self_consistent` (bool, default: `false`)
If true, iterate the diagonal self-energy to self-consistency:
E_n^{(i+1)} = diag(Hв‚Ђ + V_H) + Re ОЈ_xc(E_n^{(i)}).
Not compatible with `use_ppm_sigma = true`.

## 4. Screening

### `screening_method` (str, default: `"minimax"`)
Method for computing П‡в‚Ђ в†’ W:
- **`"minimax"`**: Laplace minimax quadrature for the imaginary-time representation
  of П‡в‚Ђ. Approximates 1/(E_c в€’ E_v) в‰€ ОЈ_l w_l exp(в€’П„_l (E_c в€’ E_v)) with O(ln R)
  nodes where R = E_bw/E_gap. This is the production method.

### `minimax_target_error` (float, default: `1e-6`)
Lв€ћ error target for the minimax rational approximation of 1/x on [E_gap, E_bw].

### `minimax_max_nodes` (int, default: `64`)
Maximum number of quadrature nodes for the minimax fit.

### `minimax_energy_reference` (str, default: `"midgap"`)
Energy reference for the minimax window construction. Options: `"midgap"`, `"vbm"`,
or a numeric value in Ry. Shifts the energy origin to center the minimax interval,
improving numerical stability.

## 5. Finite-Size (Head) Corrections

For periodic systems (sys_dim в‰Ґ 1), the q=0 Coulomb divergence is handled by:
1. Setting V(G=0) = 0 in the body
2. Computing mini-BZ-averaged heads `вџЁv(q)вџ©` and `вџЁW(q)вџ© = вџЁv/(1в€’vВ·q^T S q)вџ©`
3. Adding the head as a rank-1 correction: `(wcoul0/О©) |О¶(G=0)вџ©вџЁО¶(G=0)|`

### `wcoul0_source` (str, default: `"s_tensor"`)
Source for the qв†’0 head values:
- **`"s_tensor"`** (default): Compute the frequency-dependent S(П‰) tensor from
  dipole matrix elements (dipole.h5), then evaluate `wcoul0(П‰) = вџЁv/(1в€’vВ·q^T S(П‰) q)вџ©`
  via Sobol QMC on the mini-BZ. This gives frequency-dependent heads, which is
  required for correct GN-PPM extraction at both П‰=0 and П‰=iП‰_p.
- **`"epshead"`**: Use the static ОµвЃ»В№(G=0,G'=0) from eps0mat.h5. Static only вЂ”
  the same head is used at all frequencies. **Debug only; gives wrong PPM at П‰в‰ 0.**

### `vhead` (float, default: `None`)
Override the bare Coulomb head v(qв†’0) in atomic units (same as printed under
`FINITE-SIZE CORRECTIONS` in the output). If set together with `whead_0freq`,
bypasses `determine_wcoul0()` entirely. Useful for injecting BGW epsilon.out
head values for debugging.

### `whead_0freq` (float, default: `None`)
Override the screened Coulomb head W(qв†’0, П‰=0) in atomic units.

### `whead_imfreq` (float, default: `None`)
Override the screened Coulomb head W(qв†’0, П‰=iП‰_p) in atomic units. Only used
when `use_ppm_sigma = true`. Must be set together with `vhead` and `whead_0freq`.

## 6. GN-PPM Dynamic Self-Energy

When `use_ppm_sigma = true`, the code computes the full frequency-dependent
correlation self-energy ОЈ^c(П‰) using the Godby-Needs Generalized Plasmon Pole
model. The PPM represents W^c(П‰) = W(П‰) в€’ V as:

```
W^c_q(Ој,ОЅ,П‰) = 2 B_q(Ој,ОЅ) О©_q(Ој,ОЅ) / (П‰ВІ в€’ О©_qВІ(Ој,ОЅ))
```

where B and О© are extracted elementwise from W^c(0) and W^c(iП‰_p) via:
```
О© = П‰_p в€љ(W^c(iП‰_p) / (W^c(0) в€’ W^c(iП‰_p)))
B = в€’ВЅ W^c(0) О©
```

The self-energy is then evaluated via minimax-windowed CTSP (Complex-Time
Shredded Propagator) integration in imaginary time, decomposed into two branches:
```
ОЈвЃє(П‰) = в€’(1/N_k) ОЈ_{v,q} MВІ B/(П‰ + h_v + О©)    [occupied/valence branch]
ОЈвЃ»(П‰) = в€’(1/N_k) ОЈ_{c,q} MВІ B/(П‰ в€’ E_c в€’ О©)    [unoccupied/conduction branch]
```

### `use_ppm_sigma` (bool, default: `false`)
Enable the GN-PPM dynamic ОЈ^c(П‰) pipeline. Requires `do_screened = true`.
When enabled, replaces the static COH with the frequency-integrated result.

### `ppm_omega_p` (float, default: `2.0` Ry)
Imaginary frequency probe for the GN-PPM extraction. W is evaluated at П‰=0
and П‰=iП‰_p to determine the PPM parameters. Must match the second frequency
in BGW's epsilon (which is 2.0 Ry = 27.2 eV by default).

### `ppm_fallback_omega` (float, default: `2.0` Ry)
Fallback plasmon frequency for GN modes where the extraction gives unphysical
(negative or imaginary) О©. Used when `ppm_invalid_mode = "fixed_2ry"`.

### `ppm_invalid_mode` (str, default: `"static_limit"`)
Policy for invalid GN modes (where О©ВІ < 0 or the fit is unphysical):
- **`"static_limit"`**: Exclude invalid modes from the dynamic sum. Add a static
  correction: ОЈ_invalid = ОЈ_{occ}(G_occ, W^c_invalid(0)) в€’ ВЅ ОЈ_{all}(I, W^c_invalid(0)).
- **`"fixed_2ry"`**: Replace О© with `ppm_fallback_omega` for invalid modes.

### `fermi_reference` (str, default: `"midgap"`)
Reference energy for the П‰ grid and the ОЈВ± branch splitting:
- **`"midgap"`**: E_F = (VBM + CBM)/2. Band energies become E_c = E_{ck} в€’ E_F в‰Ґ 0
  and h_v = E_F в€’ E_{vk} в‰Ґ 0. The П‰ grid is centered on E_F.
- **`"vbm"`**: E_F = VBM.

## 7. Frequency Grid & Windowing

### `sigma_omega_min_ev` (float, default: `-5.0`)
Lower bound of the ОЈ^c(П‰) output grid in eV (relative to E_F per `fermi_reference`).

### `sigma_omega_max_ev` (float, default: `5.0`)
Upper bound of the ОЈ^c(П‰) output grid in eV.

### `sigma_omega_step_ev` (float, default: `0.25`)
Spacing of the output frequency grid. n_П‰ = floor((max в€’ min)/step) + 1.

### `sigma_regularization_ev` (float, default: `0.25`)
Regularization width Оѕ (eV) for crossing windows. Controls the sine-sum
approximation to 1/x near the singularity.

The minimax method partitions the denominators of G and W into multiple windows
with different quadrature procedures:

- **Non-crossing (sign-definite) windows**: The denominator `S В± П‰` doesn't
  change sign within the window. Approximated by exponential sums (Laplace
  minimax): `1/x в‰€ ОЈ_l О±_l exp(в€’П„_l x)`. Cost: O(ln R) nodes where R is
  the dynamic range of the window.

- **Crossing windows**: The denominator passes through zero for some (band, П‰)
  combinations. Approximated by sine sums with a regularization envelope:
  `G_reg(u) в‰€ ОЈ_l w_l sin(П„_l u)`, where `u = (Sв€’П‰)/Оѕ` is the dimensionless
  energy scaled by the regularization width Оѕ. This smooths the 1/x singularity
  over a width ~Оѕ, analogous to iО· broadening. The number of sine-sum nodes
  scales as O(A) where A = (sigma_max в€’ sigma_min) / Оѕ вЂ” **linearly proportional
  to the frequency range divided by the regularization width**. Smaller Оѕ gives
  a narrower broadening but requires more quadrature nodes and is more expensive.

- **Three-window scheme**: For the conduction branch at П‰ > 0, the code
  partitions (E_A, E_B) space into a core (crossing, sine-sum), a_stripe
  (sign-definite, Laplace), and b_slab (sign-definite, Laplace). Only the
  core uses the expensive sine-sum quadrature.

Converted to Ry internally: Оѕ_Ry = Оѕ_eV / 13.606.

### `ppm_sigma_scale` (float, default: `1.0`)
Global multiplicative scale factor for ОЈ^c. For debugging/empirical adjustment.

### `ppm_sigma_flip_neg` (bool, default: `false`)
**Debug only.** If true, flip the sign of the П‰ < E_F block. Test parameter for
sign convention debugging.

## 8. Accumulation & Evaluation at E_DFT

### `sigma_omega_accumulation` (str, default: `"auto"`)
How to accumulate ОЈ_c(k,i,j,П‰) over time nodes:
- **`"auto"`**: Choose `"kij"` (in-memory) or `"kij_stream"` based on memory.
- **`"kij"`**: Accumulate the full (n_П‰, nk, nb, nb) tensor in GPU memory. Fast
  but requires n_П‰ Г— nk Г— nbВІ Г— 16 bytes.
- **`"kij_stream"`**: Stream П‰-chunks to `sigma_kij_h5_file` on disk. For very
  large grids where in-memory accumulation exceeds GPU memory.

### `sigma_at_dft_energies` (bool, default: `false`)
If true, interpolate ОЈ^c(П‰) at П‰_rel = E_DFT в€’ E_F for each band, producing
`sigC_EDFT` in the eqp0 output. This is the quantity compared to BGW's `Cor`.

### `sigma_at_dft_extrapolate` (bool, default: `false`)
If true, extrapolate ОЈ^c(П‰) outside the grid [П‰_min, П‰_max] for bands whose
E_DFT falls outside. If false, such bands get NaN.

### `sigma_debug_split_contrib` (bool, default: `false`)
If true, store ОЈвЃє (valence/occupied branch) and ОЈвЃ» (conduction/unoccupied branch)
separately in `sigma_c_plus_kij_ev` and `sigma_c_minus_kij_ev` datasets.
Auto-enabled when `sigma_freq_debug_output = true`.

## 11. K-Points

**The k-grid for the GW calculation is taken entirely from the WFN.h5 file**
(nkx Г— nky Г— nkz stored in its header). The K_POINTS block in cohsex.in does
**NOT** control the GW k-grid. Changing the K_POINTS block does not change which
k-points are used for screening, self-energy, or any physical quantity.

The K_POINTS block is used **only** for band-structure interpolation plots (the
QP energy plot output). It follows Quantum ESPRESSO `{crystal_b}` conventions:

```
K_POINTS {crystal_b}
4
  0.0000 0.0000 0.0000 80  #Gamma
  0.0000 0.5000 0.0000 80  #M
  0.3333 0.3333 0.0000 80  #K
  0.0000 0.0000 0.0000 80  #Gamma
```

The integer after each k-point (80) specifies the number of interpolation points
along each segment for the band-structure plot. It is not used by the GW engine.
