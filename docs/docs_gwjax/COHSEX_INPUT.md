# GWJAX `cohsex.in` Input Reference

GWJAX is the GW module of **LORRAX** (**Lo**w-scaling **R**eal-space **R**eal-**A**xis
e**X**cited state package). This is the exhaustive reference for every parameter
in the GWJAX input file. All parameters
are read by `gw_init.read_cohsex_input()` from a `[cohsex]` INI section.

---

## 1. Restart & ISDF Basis

### `restart` (bool, default: `true`)
If true, load precomputed ISDF tensors (V_qμν, wavefunctions, centroids) from
`tmp/isdf_tensors_<N>.h5`. If false, recompute from scratch (slow: FFT + Cholesky
fitting of zeta_q). Must be false on first run or when `nband` changes.

**Restart file contents** (`tmp/isdf_tensors_<N>.h5`):
| Dataset | Shape | Type | Meaning |
|---------|-------|------|---------|
| `psi_full_y` | (nk, nband, nspinor, n_rmu) | complex128 | Wavefunctions at ISDF centroids (Y-layout) |
| `enk_full` | (nk, nband) | float64 | DFT eigenvalues (Ry) |
| `V0_noG0_munu` | (n_rmu, n_rmu) | complex128 | Bare Coulomb V(q=0) without G=0 head |
| `G0_mu_nu` | (n_rmu,) | complex128 | G=0 centroid overlap vector ζ_μ(G=0) |

### `centroids_file` (str, default: `"centroids_frac.txt"`)
Path to ISDF centroid coordinates in fractional crystal coordinates. One line per
centroid (3 floats). Generated by `python -m centroid.kmeans_isdf`. The number
of centroids N_μ determines the ISDF basis size; recommended 8×(nval+ncond) for
debugging, 12× for production.

### `use_chunked_isdf` (bool, default: `true`)
Use memory-efficient chunked ISDF fitting (processes r-space in chunks). Required
for large systems where the full FFT grid doesn't fit in GPU memory.

### `memory_per_device_gb` (float, default: `0.0`)
GPU memory budget in GB. 0.0 = auto-detect (90% of available GPU memory via
`jax.memory_stats()` or `nvidia-smi`). The chunk solver uses this to determine
r-chunk, band-chunk, and q-chunk sizes. See `MEMORY_MODEL.md` for formulas.

### `zeta_ridge_eps` (float, default: `0.0`) — opt-in, charge-only
Relative Tikhonov ridge for the ζ-fit. `0.0` (default) = OFF, bit-identical to
the historical LSQ fit. When `> 0` the fit solves the SPD normal equations
`(C² + ε_q²I) ζ = C Z` with `ε_q = zeta_ridge_eps · λ̂_max(C_q)` per q
(deterministic power-iteration λ̂, no eigh) — the spectral filter
`f_ε(λ) = λ/(λ²+ε²)`, i.e. the "cleaned ζ" of the F-scheme study. RELATIVE
scaling by design so the knob transfers across system sizes. Charge channel
only: rejected with `bispinor = true` (transverse indefinite-CCT follow-up).
Effective solver conditioning: `cond(C²+ε²I) ≈ min(cond(C)², 1/eps_rel²)` —
the crossover ε_rel* = cond(C)^{-1/2} (~1e-4 at fixture scale). A/B results
(reports/zeta_ridge_ab_2026-07-17/): NOT physically inert on GW Σ — drifts
are ~1e-3 relative of Σ (meV–100 meV absolute); V/W-tile q-covariance
improves ~8× at 1e-4 on Si. Diagnostic/experiments knob; do not enable in
production comparisons without an A/B against `zeta_ridge_eps = 0`.

## 2. Band Window

The GW calculation partitions bands into windows defined by four edges b0..b4.
The number of electrons `nelec` is read from the WFN file.

### `nval` (int, default: `5`)
Number of **valence** (occupied) bands in the sigma evaluation window. Sets
`b1 = nelec − nval`. These are the highest occupied bands for which Σ is computed.

### `ncond` (int, default: `5`)
Number of **conduction** (unoccupied) bands in the sigma evaluation window. Sets
`b3 = nelec + ncond`. These are the lowest unoccupied bands for which Σ is computed.

### `nband` (int, default: `100`)
Total number of bands loaded from the WFN for screening (χ₀) and self-energy sums.
Sets `b4 = nband`. Must match `number_bands` in BGW's epsilon.inp and sigma.inp
for fair comparison. The WFN file must contain at least this many bands.

**Band edge summary** (all 0-indexed):
```
b0 = 0          (bottom of full range)
b1 = nelec − nval   (bottom of sigma window, within valence)
b2 = nelec          (Fermi level: top of occupied)
b3 = nelec + ncond  (top of sigma window, within conduction)
b4 = nband          (top of full range for screening)
```

**Wavefunction slices** built from these:
| Slice | Range | Used for |
|-------|-------|----------|
| `v_slice` | b0 : b2 | Valence (occupied) bands in χ₀ |
| `c_slice` | b2 : b4 | Conduction (unoccupied) bands in χ₀ |
| `l_slice` | b0 : b3 | Sigma window bands (for SX Green's function) |
| `coh_slice` | b0 : b4 | All bands (for COH resolution of identity and PPM G) |

### `sys_dim` (int, default: `2`)
System dimensionality controlling the Coulomb truncation:
- **0**: Molecule (Wigner-Seitz box truncation). V(G=0) is finite. The q=0 head
  of W equals V (no mini-BZ averaging needed).
- **2**: 2D slab (cell slab truncation). V(q+G) = 8π/|q+G|² × (1 − e^{−z_c|q+G|_xy} cos(q_z z_c))
  divided by cell volume. V(G=0) is set to zero; the head is injected separately
  via mini-BZ-averaged wcoul0.

Corresponds to BGW's `cell_box_truncation` (0) or `cell_slab_truncation` (2).

## 3. Calculation Mode

### `x_only` (bool, default: `false`)
If true, compute only bare exchange Σ_X = −(1/N_k) Σ_q G_{occ,k−q} V_q. No
screening, no correlation. Mutually exclusive with `do_screened = true`.

### `do_screened` (bool, default: `true`)
If true, compute the screened Coulomb interaction W from χ₀ via the Dyson equation
W = V(I − Vχ₀)⁻¹. The static COHSEX self-energy is then:
```
Σ_SX = −(1/N_k) Σ_q G_{occ,k−q} W_q
Σ_COH = −(1/2N_k) Σ_q I_{k−q} (W_q − V_q)
```
If false, W = V (bare exchange only).

### `bispinor` (bool, default: `false`)
If true, use 4-component Dirac bispinor wavefunctions with γ⁰ Coulomb vertex
factors: Σ_{ab} = γ⁰_{aa} γ⁰_{bb} G_{ab} V, where γ⁰ = diag(1,1,−1,−1).
If false, use 2-component Pauli spinors (standard for spin-orbit coupling).

### `qp_solver` (str, default: `"auto"` → `"one_shot_dft"`)
How QP energies are extracted from Σ — orthogonal to the Σ ansatz
(`compute_mode`). Three states, each naming a standard method:

- **`"one_shot_dft"`** (the default): textbook G0W0. Σ is built once from the
  DFT inputs and *everything* is evaluated at E_DFT — the eqp0/eqp1 text
  outputs (at-DFT Newton + Z-linearization, as always) AND the
  QSGW-symmetrised Σ_xc whose eigh produces `E_qp_ry` /
  `qp_wfn_rotations.h5` / `WFN_qp.h5`. No iteration of any kind.
- **`"fixed_point"`**: one-shot Σ + diagonal on-shell solve
  E = h0 + Re Σ(E) for the QSGW-build evaluation energies (eigenvalue-only;
  Σ is never rebuilt). Dynamic modes (`gn_ppm`/`hl_ppm`) only — rejected at
  config validation for static modes (no ω-grid to solve on).
  `sigma_at_dft_extrapolate` is a sub-knob of this state.
- **`"self_consistent"`**: full QSGW loop — Σ rebuilt each iteration from
  rotated ψ + the previous iteration's E (ζ/V_q are *not* refit; χ₀→W is
  re-solved every iteration). Works for all modes (COHSEX and GN-PPM
  verified end-to-end). Loop knobs: the `sc_*` keys below.

`"auto"` resolves: `self_consistent = true` → `"self_consistent"`, else
`"one_shot_dft"`. Validation: `fixed_point`/`self_consistent` ×
`sigma_omega_accumulation = kij_stream` is rejected (streamed Σ_c(ω) leaves
no in-memory ω-tensor for the on-shell solve / QSGW rebuild).

eqp0.dat / eqp1.dat keep the same formula in all three states; only the
provenance of Σ changes under `self_consistent` (converged Σ, still
evaluated at E_DFT).

### `self_consistent` (bool, default: `false`) — DEPRECATED
Deprecated alias (2026-07-08) for `qp_solver = self_consistent`; still
honored via `qp_solver = auto` resolution. Set `qp_solver` instead.

### `sc_max_iter` (int, default: `20`)
Maximum self-consistency iterations (`qp_solver = self_consistent` only).
The deprecated `LORRAX_SC_MAX_ITER` env var overrides it (a note is printed).

### `sc_tol_ev` (float, default: `1e-4`)
Convergence threshold on the RMS ΔE (eV) between consecutive iterations'
QP eigenvalues. Env override: `LORRAX_SC_TOL_EV`.

### `sc_accelerator` (str, default: `"rcrop"`)
- **`"rcrop"`**: Anderson-style restart-CROP acceleration — required for QSGW
  on dense band manifolds (the fixed-point Jacobian typically has a 2-cycle
  direction). Makes TWO pipeline calls per accelerator iteration
  (trial + residual).
- **`"linear"`**: plain α-mixing with damping `sc_mixing` (diagnostic).

Env override: `LORRAX_SC_ACCEL`.

### `sc_history_depth` (int, default: `5`)
rCROP history depth (m=5 is BGW's QSGW default). Env override:
`LORRAX_SC_DEPTH`.

### `sc_mixing` (float, default: `1.0`)
Linear damping coefficient α (`sc_accelerator = linear` only). Env override:
`LORRAX_SC_MIXING`.

### `sc_dump_dir` (str, default: `""`)
If set, dump the per-iteration QP-energy history to
`<sc_dump_dir>/e_history_kn_ev.npy`. Env override: `LORRAX_SC_DUMP_DIR`.

## 4. Screening

### `screening_method` (str, default: `"minimax"`)
Method for computing χ₀ → W:
- **`"minimax"`**: Laplace minimax quadrature for the imaginary-time representation
  of χ₀. Approximates 1/(E_c − E_v) ≈ Σ_l w_l exp(−τ_l (E_c − E_v)) with O(ln R)
  nodes where R = E_bw/E_gap. This is the production method.

### `minimax_target_error` (float, default: `1e-6`)
L∞ error target for the minimax rational approximation of 1/x on [E_gap, E_bw].

### `minimax_max_nodes` (int, default: `64`)
Maximum number of quadrature nodes for the minimax fit.

### `minimax_energy_reference` (str, default: `"midgap"`)
Energy reference for the minimax window construction. Options: `"midgap"`, `"vbm"`,
or a numeric value in Ry. Shifts the energy origin to center the minimax interval,
improving numerical stability.

## 5. Finite-Size (Head) Corrections

For periodic systems (sys_dim ≥ 1), the q=0 Coulomb divergence is handled by:
1. Setting V(G=0) = 0 in the body
2. Computing mini-BZ-averaged heads `⟨v(q)⟩` and `⟨W(q)⟩ = ⟨v/(1−v·q^T S q)⟩`
3. Adding the head as a rank-1 correction: `(wcoul0/Ω) |ζ(G=0)⟩⟨ζ(G=0)|`

### `wcoul0_source` (str, default: `"s_tensor"`)
Source for the q→0 head values:
- **`"s_tensor"`** (default): Compute the frequency-dependent S(ω) tensor from
  dipole matrix elements (dipole.h5), then evaluate `wcoul0(ω) = ⟨v/(1−v·q^T S(ω) q)⟩`
  via Sobol QMC on the mini-BZ. This gives frequency-dependent heads, which is
  required for correct GN-PPM extraction at both ω=0 and ω=iω_p.
- **`"epshead"`**: Use the static ε⁻¹(G=0,G'=0) from eps0mat.h5. Static only —
  the same head is used at all frequencies. **Debug only; gives wrong PPM at ω≠0.**

### `vhead` (float, default: `None`)
Override the bare Coulomb head v(q→0) in atomic units (same as printed under
`FINITE-SIZE CORRECTIONS` in the output). If set together with `whead_0freq`,
bypasses `determine_wcoul0()` entirely. Useful for injecting BGW epsilon.out
head values for debugging.

### `whead_0freq` (float, default: `None`)
Override the screened Coulomb head W(q→0, ω=0) in atomic units.

### `whead_imfreq` (float, default: `None`)
Override the screened Coulomb head W(q→0, ω=iω_p) in atomic units. Only used
when `use_ppm_sigma = true`. Must be set together with `vhead` and `whead_0freq`.

## 6. GN-PPM Dynamic Self-Energy

When `use_ppm_sigma = true`, the code computes the full frequency-dependent
correlation self-energy Σ^c(ω) using the Godby-Needs Generalized Plasmon Pole
model. The PPM represents W^c(ω) = W(ω) − V as:

```
W^c_q(μ,ν,ω) = 2 B_q(μ,ν) Ω_q(μ,ν) / (ω² − Ω_q²(μ,ν))
```

where B and Ω are extracted elementwise from W^c(0) and W^c(iω_p) via:
```
Ω = ω_p √(W^c(iω_p) / (W^c(0) − W^c(iω_p)))
B = −½ W^c(0) Ω
```

The self-energy is then evaluated via minimax-windowed CTSP (Complex-Time
Shredded Propagator) integration in imaginary time, decomposed into two branches:
```
Σ⁺(ω) = −(1/N_k) Σ_{v,q} M² B/(ω + h_v + Ω)    [occupied/valence branch]
Σ⁻(ω) = −(1/N_k) Σ_{c,q} M² B/(ω − E_c − Ω)    [unoccupied/conduction branch]
```

### `use_ppm_sigma` (bool, default: `false`)
Enable the GN-PPM dynamic Σ^c(ω) pipeline. Requires `do_screened = true`.
When enabled, replaces the static COH with the frequency-integrated result.

### `ppm_omega_p` (float, default: `2.0` Ry)
Imaginary frequency probe for the GN-PPM extraction. W is evaluated at ω=0
and ω=iω_p to determine the PPM parameters. Must match the second frequency
in BGW's epsilon (which is 2.0 Ry = 27.2 eV by default).

### `ppm_fallback_omega` (float, default: `2.0` Ry)
Fallback plasmon frequency for GN modes where the extraction gives unphysical
(negative or imaginary) Ω. The fitted pole is only used when
`ppm_invalid_mode = "2ry"`, but the fit always bakes it in (`B = −½·W^c(0)·Ω`).

### `ppm_invalid_mode` (str, default: `"static_limit"`)
Policy for invalid GN modes (Ω² < 0), = BGW `invalid_gpp_mode`
(implemented 2026-07-08; validated vs BGW modes 0/2/3 on Si 4×4×4 —
see `reports/bgw_invalid_mode_refs_2026-07-08/`):
- **`"static_limit"`** (BGW mode 3 = BGW default): exclude invalid modes from
  the dynamic pole sum and add the analytic ω-independent static-COHSEX term
  Σ_invalid = Σ_occ(G_occ, W^c_inv(0)) − ½ Σ_RI(I, W^c_inv(0)).
- **`"zero"`** (BGW mode 0): drop invalid modes entirely.
- **`"2ry"`** (BGW mode 2): keep the fit's `ppm_fallback_omega` pole.
- `"infinity"` is accepted as an alias of `"static_limit"`; `"imaginary"`
  (BGW mode 1) is unsupported.

### `fermi_reference` (str, default: `"midgap"`)
Reference energy for the ω grid and the Σ± branch splitting:
- **`"midgap"`**: E_F = (VBM + CBM)/2. Band energies become E_c = E_{ck} − E_F ≥ 0
  and h_v = E_F − E_{vk} ≥ 0. The ω grid is centered on E_F.
- **`"vbm"`**: E_F = VBM.

## 7. Frequency Grid & Windowing

### `sigma_omega_min_ev` (float, default: `-5.0`)
Lower bound of the Σ^c(ω) output grid in eV (relative to E_F per `fermi_reference`).

### `sigma_omega_max_ev` (float, default: `5.0`)
Upper bound of the Σ^c(ω) output grid in eV.

### `sigma_omega_step_ev` (float, default: `0.25`)
Spacing of the output frequency grid. n_ω = floor((max − min)/step) + 1.

### `sigma_regularization_ev` (float, default: `0.25`)
Regularization width ξ (eV) for crossing windows. Controls the sine-sum
approximation to 1/x near the singularity.

The minimax method partitions the denominators of G and W into multiple windows
with different quadrature procedures:

- **Non-crossing (sign-definite) windows**: The denominator `S ± ω` doesn't
  change sign within the window. Approximated by exponential sums (Laplace
  minimax): `1/x ≈ Σ_l α_l exp(−τ_l x)`. Cost: O(ln R) nodes where R is
  the dynamic range of the window.

- **Crossing windows**: The denominator passes through zero for some (band, ω)
  combinations. Approximated by sine sums with a regularization envelope:
  `G_reg(u) ≈ Σ_l w_l sin(τ_l u)`, where `u = (S−ω)/ξ` is the dimensionless
  energy scaled by the regularization width ξ. This smooths the 1/x singularity
  over a width ~ξ, analogous to iη broadening. The number of sine-sum nodes
  scales as O(A) where A = (sigma_max − sigma_min) / ξ — **linearly proportional
  to the frequency range divided by the regularization width**. Smaller ξ gives
  a narrower broadening but requires more quadrature nodes and is more expensive.

- **Three-window scheme**: For the conduction branch at ω > 0, the code
  partitions (E_A, E_B) space into a core (crossing, sine-sum), a_stripe
  (sign-definite, Laplace), and b_slab (sign-definite, Laplace). Only the
  core uses the expensive sine-sum quadrature.

Converted to Ry internally: ξ_Ry = ξ_eV / 13.606.

### `ppm_sigma_scale` (float, default: `1.0`)
Global multiplicative scale factor for Σ^c. For debugging/empirical adjustment.

### `ppm_sigma_flip_neg` (bool, default: `false`)
**Debug only.** If true, flip the sign of the ω < E_F block. Test parameter for
sign convention debugging.

## 8. Accumulation & Evaluation at E_DFT

### `sigma_omega_accumulation` (str, default: `"auto"`)
How to accumulate Σ_c(k,i,j,ω) over time nodes:
- **`"auto"`**: Choose `"kij"` (in-memory) or `"kij_stream"` based on memory.
- **`"kij"`**: Accumulate the full (n_ω, nk, nb, nb) tensor in GPU memory. Fast
  but requires n_ω × nk × nb² × 16 bytes.
- **`"kij_stream"`**: Stream ω-chunks to `sigma_kij_h5_file` on disk. For very
  large grids where in-memory accumulation exceeds GPU memory.

### `sigma_at_dft_energies` (bool, default: `false`) — DEPRECATED
Deprecated alias (2026-07-08) for `qp_solver = one_shot_dft` — which is now
the default. (The at-DFT interpolation of Σ^c(ω), producing the `sigC`
column compared to BGW's `Cor`, runs unconditionally in every dynamic mode;
this key's intended meaning — authoritative at-DFT QP evaluation for the
eigh-family outputs too — is exactly `qp_solver = one_shot_dft`.) Remove it
from input files; set `qp_solver` instead.

### `sigma_at_dft_extrapolate` (bool, default: `false`)
Sub-knob of `qp_solver = fixed_point` (ignored in the other two states).
If true, bands whose E_DFT falls outside the ω-grid [ω_min, ω_max] get a
scissor extrapolation (fitted on in-grid bands) instead of the on-shell
solve; if false, such bands fall back to E_DFT.

### `sigma_debug_split_contrib` (bool, default: `false`)
If true, store Σ⁺ (valence/occupied branch) and Σ⁻ (conduction/unoccupied branch)
separately in `sigma_c_plus_kij_ev` and `sigma_c_minus_kij_ev` datasets.
Auto-enabled when `sigma_freq_debug_output = true`.

## 9. Output Files

### `wfn_file` (str, default: `"WFN.h5"`)
Input wavefunction file in BerkeleyGW HDF5 format. Contains k-points, eigenvalues,
plane-wave coefficients, cell geometry, FFT grid.

### `output_file` (str, default: `"eqp0_noqsym.dat"`)
Main output file. Format depends on whether PPM is active:

**With `use_ppm_sigma = true` and `sigma_at_dft_energies = true`:**
```
# Sigma output (all in eV)
# sigXC_EDFT = sigX + sigC_EDFT

k-point 0:
---...---
n=0   sigX=  -18.376  sigC_EDFT=   2.559+0.007i  sigXC_EDFT=  -15.817+0.007i  VH=  459.730+0.000i
```

**Without PPM (static COHSEX):**
```
  kx  ky  kz  nband
  spin  band  E_DFT(eV)  E_QP(eV)
```

### `sigma_omega_h5_file` (str, default: `""`)
HDF5 file for the full Σ(ω) tensor. Written when `use_ppm_sigma = true`.

**Datasets:**
| Dataset | Shape | Type | Meaning |
|---------|-------|------|---------|
| `omega_ev` | (n_ω,) | float64 | Frequency grid (eV) |
| `sigma_c_kij_ev` | (n_ω, nk, nb, nb) | complex128 | Σ^c(ω) in eV |
| `sigma_c_plus_kij_ev` | (n_ω, nk, nb, nb) | complex128 | Σ⁺ branch (if split) |
| `sigma_c_minus_kij_ev` | (n_ω, nk, nb, nb) | complex128 | Σ⁻ branch (if split) |
| `sigma_c_invalid_static_kij_ev` | (nk, nb, nb) | complex128 | Invalid-mode static correction |
| `sigma_sx_kij_ev` | (nk, nb, nb) | complex128 | Bare exchange Σ_X (ω-independent) |
| `sigma_total_kij_ev` | (n_ω, nk, nb, nb) | complex128 | Σ_X + Σ^c(ω) |
| `hartree_kij_ev` | (nk, nb, nb) | complex128 | Hartree V_H |

Here nb = b3 − b0 (sigma window size), nk = nkx×nky×nkz.

### `sigma_freq_debug_output` (bool, default: `false`)
If true, write `sigma_freq_debug.dat` with per-band decomposition. Also forces
`sigma_debug_split_contrib = true`.

**`sigma_freq_debug.dat` columns** (tab-separated, all in eV):
| Col | Name | Meaning |
|-----|------|---------|
| 0 | k | k-point index (0-indexed) |
| 1 | n | Band index (0-indexed; physical = n+1) |
| 2 | Edft-Ef | E_DFT − E_F(midgap) |
| 3 | E_dft | Absolute DFT eigenvalue |
| 4 | kin_ion | Diagonal kinetic + ionic potential matrix element (T + V\_ion + V\_NL)\_{nn} from kin\_ion.h5 (eV) |
| 5 | sex_0 | Static Σ_SX at ω=0 |
| 6 | coh_0 | Static Σ_COH at ω=0 |
| 7 | x_bare | Bare exchange Σ_X |
| 8 | sig_c(0) | PPM Σ^c at ω=0 |
| 9 | sig_c+(w) | Σ⁺ at ω=E_DFT (NaN if out of grid) |
| 10 | sig_c-(w) | Σ⁻ at ω=E_DFT (NaN if out of grid) |
| 11 | sig_c_invld(0) | Invalid-mode static correction |
| 12 | sig_c(Edft) | Total Σ^c at ω=E_DFT (**= sigC_EDFT**) |

## 10. Debug Flags

### `write_w_copies_debug` (bool, default: `false`)
Write W(0) and W(iω_p) snapshots to `w_copies_debug.h5`. Useful for verifying
that the minimax W matches the static screening W.

**`w_copies_debug.h5` datasets:**
| Dataset | Shape | Meaning |
|---------|-------|---------|
| `W0_screen_q000_munu` | (n_rmu, n_rmu) | W(0) from static screening at q=0 |
| `W0_ppm_q000_munu` | (n_rmu, n_rmu) | W(0) from PPM minimax at q=0 |
| `Wiwp_ppm_q000_munu` | (n_rmu, n_rmu) | W(iω_p) from PPM minimax at q=0 |

### `write_no_head_vw` (bool, default: `false`)
Store V and W without the G=0 head in the restart file. Used by the ISDF SoS
debug script to reconstruct the body separately from the head.

### `debug_hartree` (bool, default: `false`)
Print diagnostics for the Hartree potential computation.

### `ppm_sigma_debug_static_norm` (bool, default: `false`)
Compare the PPM Σ^c(ω=0) against static COH computed from W^c(0) = −2B/Ω.
Prints the max and relative differences.

### `ppm_static_cohsex_check` (bool, default: `false`)
**Verification test.** Sets all band energies to zero and ω=[0], then checks that
Σ^c(0) = Σ_cor = (SX−X) + COH. The reference is Σ_cor (not just COH), since
Σ^c = Σ_xc − Σ_X = (Σ_SX − Σ_X) + Σ_COH.

### `sigma_debug_quadrature` (bool, default: `false`)
Print per-window minimax quadrature fit errors (samples the approximation on a
dense grid and reports L∞ error).

## 11. K-Points

**The k-grid for the GW calculation is taken entirely from the WFN.h5 file**
(nkx × nky × nkz stored in its header). The K_POINTS block in cohsex.in does
**NOT** control the GW k-grid. Changing the K_POINTS block does not change which
k-points are used for screening, self-energy, or any physical quantity.

The K_POINTS block is used **only** for band-structure interpolation plots (the
QP energy plot output). It follows Quantum ESPRESSO `{crystal_b}` conventions:

